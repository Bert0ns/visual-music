# Implementation Plan: ProjectM Cross-Platform Visualizer

## Overview

Building a high-performance cross-platform music visualizer using Flutter and the ProjectM (MilkDrop) C++ engine. The plan breaks down the complex FFI bindings, audio capture logic, and immersive UI into vertically sliced tasks, ensuring we build and test the most foundational pieces (the C++ bridge and rendering pipeline) before moving on to UI and advanced OS-level audio capture.

## Architecture Decisions

- **FFI for ProjectM**: C++ ProjectM library will be compiled natively for each platform. Dart FFI will invoke C++ functions, which will render into an OpenGL frame buffer and pass the texture ID to Flutter's `Texture` widget.
- **Pure C++ Audio Capture**: To ensure zero-latency audio analysis and avoid FFI overhead, audio capture will be handled entirely in C++ using a lightweight library like `miniaudio` (rather than capturing in Dart and passing bytes across the bridge).
- **Preset Storage**: Presets will be bundled as a single `presets.zip` asset. On first launch, an Isolate will extract them and build a local SQLite database for instant searching and managing Heart/Ban metadata.
- **State Management**: Riverpod will be used to manage UI state (like/ban, current preset, audio source) without blocking the rendering thread.

## Risks and Mitigations

| Risk                           | Impact | Mitigation                                                                                                                                                     |
| ------------------------------ | ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **FFI Memory Leaks**           | High   | Use Dart `NativeFinalizer` to ensure C++ objects are destroyed when Dart objects are garbage collected.                                                        |
| **OpenGL Context Sharing**     | High   | Flutter's texture registry requires strict thread affinity. We will ensure all OpenGL rendering in C++ is synchronized with Flutter's raster thread.           |
| **Android System Audio Block** | High   | Many apps (Netflix, Apple Music) use `FLAG_SECURE` which blocks system audio capture. Mitigate by providing an immediate, seamless fallback to the Microphone. |
